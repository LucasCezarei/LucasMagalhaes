    Instalações

    1) Para a instalação dos arquivos se encontram no seguinte repositório:

    Arquivo: 

    jdk-8u221-windows-x64 (JAVA)

    Instalar na pasta C:\Program Files\Java





    Instale o arquivo: 

    blaze-advisor_7.5.7500.v11760_Windows (Blaze)

    No meu caso, ele está na seguinte pasta: C:\Blaze.





    2) Criar uma pasta em C:\Blaze e copiar extrair o arquivo eclipse-jee-oxygen-3a-win32-x86_64.zip para dentro desta pasta.








    3)Renomear a pasta eclipse-jee-oxygen-3a-win32-x86_64 para eclipse para facilitar a localização.








    4)Executar eclipse.exe que está dentro da pasta eclipse.








    Aviso: Escolher aonde ira ficar a pasta workspace do eclipse.

    No meu caso, escolhi a própria pasta do eclipse. 

    C:\Blaze\projetos\eclipse01\eclipse-workspace




    5)Instalar o plug-in do Blaze: após abrir o eclipse e escolher o workspace, devemos preparar o eclipse para suportar o Blaze.

    • No eclipse, ir em: Help - Install new software.







    • Add - local.







    • 6)Ir na pasta C:\Blaze\Advisor75\EclipsePluginUpdateSite







    • 7)Depois de selecionar a pasta, digitar Blaze Advisor e clicar em ok.







    • 8)Selecionar Blaze Adivisor e Blaze Advisor Optional. Ir clicando em next até terminar a instalação e reiniciar o eclipse.







    Aviso: Para utilizar o plug-in do Blaze:

    • Ao reiniciar, Colocar a licença do Blaze.

      Window - Perspective - Open Perspective - Other.








      • blaze Advisor - Open.








      Criando Repositório

      Todo nosso ambiente será preparado na pasta 

      C:\Blaze\projetos

      1) Criar uma pasta com o nome escolhido para o seu repositorio em C:\Blaze\projetos








      2) Neste tutorial com exemplo, iremos utilizar o sistema de versionamento do Blaze (bvs) privado.

      Aviso: Este sistema de versionamento funciona com base em duas partes, uma denominada Central e outra Admin. A parte central mantém um histórico de tudo que for feito no Blaze e a parte Admin é o usuário criado para gerenciar o repositório/projeto.

      Crie dentro da pasta do seu repositorio, as seguintes pastas:

        
                            
      • bvs-central
        • 
                            
      • bvs-admin
        • 
                            
      • Workspaces
        •








      3)Iniciando o repositório no eclipse:

      • Após criar as pastas necessárias volte ao projeto vá em:

        • Repository - New Repository [Novo Repositório]








        • 4)Escolher um [NOME] para o repositório. No meu caso: RepositorioLucas

        • >Escolher um Repository Type [Tipo de Repositório]: repositório de Arquivos (Controle de Versão de BVS - Privado).

        • >Ainda nesta janela, fazer referenciar as pastas bvs-admin e bvs-central que foi criada.

        • >Assim comom nas imagens abaixo, você deverá referenciar a pasta bvs-central primeiro em seguida a pasta bvs-admin.

        • >Colocar admin como usuário nas duas opções. (Não é necessario a utilização de senha neste caso).











      Criando um projeto

      A estrutura do nosso projeto será a seguinte: 

      > Technical - Business - Testing - Decision Services

      Cada parte será um subprojeto e terá funções específicas.

      Para iniciar, iremos criar a parte Technical do projeto na aba:

      • File - New - Project








      • Blaze Advisor - Rule Project - Next








      • Escolher o repositório que criamos next








      • Selecionar o repositório e colocar o nome do projeto: Technical Library








      Business Object Model (BOM)

      1) Será utilizado um arquivo.jar para importar as classes e as variáveis que será utilizado neste exemplo.

      Em seu projeto Blaze você deve preparar um rol de variáveis que irá trabalhar.

      Neste projeto iremos trabalhar com as seguintes variáveis:


      • Variáveis de Entrada:
        • RENDA - SEXO - IDADE - ESTADO_CIVIL - NUMERO_CONDUTORES

      • Variáveis de Saída:
        • DIARIA - VALOR_AUTOMOVEL - VALOR_CAUCAO - CLIENT_TYPE - DECISION

            RMA [Rule Maintenance Application]

            Antes de iniciar as configurações, vá em:

            C:\Blaze\projetos e crie um pasta chamada adb.

            1) Criar uma pasta com o nome de Deploy dentro do projeto Technical. Clicar com o botão direito sobre a pasta Technical dentro do projeto principal [Regras...].

            • New - Folder - Name: Deploy.

          • 2)Clicar com o botão direito do mouse sobre a pasta criada [Deploy]. 

            • New - Deployment Entities - Rule Maintenance Application Definition - Entity Name: RMADefinition - Finish.








        • 3) Na aba RMADefinition: 

          • Application Title [Título do Aplicativo]: Regras de Aluguel de Carros

          • Habilitar as seguintes opções:

            • Download Rules Command








            • 4)Ir na aba navigate, escolher as pastas que irão aparecer no RMA. As pastas que não vão aparecer devem ficar do lado direito.








            • 5)Ir na aba applets e marcar as opções:

              • Allow Editing of Table Name.
                • 
                            
              • Display Branch Condition.

            • 6) Ir na aba Life Cycle Management, marcar todas as opções e referenciar a pasta adb criada no início deste tópico, como mostra a imagem a seguir:

            Gerar RMA

            • 1)Clicar no projeto principal [Regras...] e ir em:

            • Project - Rule Maintenance Application - Generate Rule Maintenance Application.

            • Este comando irá abrir uma janela pop-up chamada Gerador de Aplicativo.

            • Aviso: O Blaze (eclipse) já reconhece o arquivo de configurações que criamos, o RMADefinition. Este arquivo configura o deploy do projeto para o RMA WEB.

            • Deixar a opção de Deploy como:

              • Web Application.








            • 2) Clicar em next, devemos selecionar o arquivo bom-Carro3.jar que criamos e anexamos ao nosso projeto e, por fim, clicar em concluir.









            • O RMA irá abrir automaticamente no navegador Internet Explorer e irá utilizar o local host [http://127.0.0.1:8080] para gerar a interface.

              Aviso: Você pode copiar o link e trocar de navegador.

            • Se você seguiu todos os passos até aqui, basta digitar admin e admin na senha para que você acesse a interface do RMA.

            • Você acaba de acessar o RMA.
